# 舒痛 Demo

这是一个面向成年人的本地疼痛记录、就医准备与低风险行动建议演示。它不诊断、不处方，也不替代急救或线下面诊。

## 离线使用

直接用浏览器打开 `shutong-app.html`。进入记录前可选择导入既往医疗信息、疼痛发作频率、持续时间与自述详情；可跳过、编辑或删除。记录、既往信息、记忆、计划与摘要仅保存在该浏览器的本地存储中。既往信息只会按用户操作写入就诊摘要，不会传给 AI 行动建议服务。

“计划”页采用两次真实 AI 请求的三步流程：选择或填写生活目标、`/api/goal` 将目标整理为清晰的非医疗化表述、`/api/plan` 再从审核行动库生成本周节奏和当天的一个小行动。服务未启动、API Key 未配置或上游失败时，页面会显示具体原因并允许重试，不会静默改用 AI 结果以外的内容。

生成计划后，“今日行动”页可记录完成、部分完成、未执行或因不适/不确定而暂停；自由文本备注只保存在本地。下一次 AI 规划只接收最近一周的完成、暂停和未执行次数汇总，用于更保守地选择行动，不会接收备注正文。

## 可选的 AI 行动建议

服务端只把模型输出映射到内置的审核行动库；浏览器不保存或发送 API Key。未配置服务端或服务不可用时，页面会使用本地安全模板。

```powershell
$env:DEEPSEEK_API_KEY='你的密钥'
node .\shutong-api-server.mjs
```

随后访问 <http://localhost:8787/shutong-app.html>。服务仅监听本机回环地址；也支持直接打开 `shutong-app.html`，它会仅向本机 `127.0.0.1:8787` 请求 AI。可用 `PORT`、`AI_PROVIDER`、`DEEPSEEK_MODEL` 或 `OPENAI_MODEL` 覆盖默认配置；选择 OpenAI 时使用 `OPENAI_API_KEY`。

## 手机上的轻量使用

无需 APK。让电脑与手机连接同一受信任的 Wi‑Fi，然后以显式局域网模式启动：

```powershell
$env:DEEPSEEK_API_KEY='你的密钥'
$env:HOST='0.0.0.0'
node .\shutong-api-server.mjs
```

在电脑运行 `ipconfig` 查看 Wi‑Fi 的 IPv4 地址；手机浏览器访问 `http://电脑IPv4地址:8787/shutong-app.html`，再使用浏览器的“添加到主屏幕”。请只在受信任的家庭/个人网络中使用此模式；结束使用后关闭服务窗口，重新启动时不设置 `HOST` 即恢复为仅本机访问。

## 公网部署（Render）

项目已包含 `render.yaml` 和访问密码保护。将项目推送到你的私有 GitHub 仓库后，在 Render 创建 **Web Service** 并连接该仓库；Render 会读取配置，提供 HTTPS 公网地址。

在 Render 的环境变量中填写以下两项机密值，绝不提交到 Git：

- `DEEPSEEK_API_KEY`：你的 DeepSeek API Key。
- `APP_ACCESS_PASSWORD`：手机访问网页时使用的强密码。

公开地址会要求浏览器输入访问密码，网页和 AI 接口均受保护。健康检查端点不含任何密钥或模型配置。

## 安全说明

紧急信号会优先中断常规流程；“不确定”会进入线下帮助路径。紧急路径进行中，行动建议生成会暂停。正式上线前仍需由当地临床治理团队维护分流规则、急救与危机资源、隐私同意和审计要求。

## 心向 ACT 心理灵活性练习

`act-app.html` 是独立的、个人本地使用的接纳承诺疗法（ACT）自助练习应用。直接打开文件即可使用本地价值行动、六项短练习和回顾功能；记录仅保存在当前浏览器。

如需 AI 协助整理困难或缩小行动，请启动本机服务：

```powershell
$env:DEEPSEEK_API_KEY='你的密钥'
node .\act-api-server.mjs
```

随后访问 <http://127.0.0.1:8788/act-app.html>；也可直接打开 `act-app.html`，它只会向本机 `127.0.0.1:8788` 发出 AI 请求。服务支持 `AI_PROVIDER=openai` 与 `OPENAI_API_KEY`，或使用 `ACT_PORT` 更换端口。心向不提供诊断、心理治疗、危机干预或医疗建议；即时支持提示位于应用设置页。
